🔢 大数值场景 ​

当数据是 [10, 50, 200] 这种小数值时，库直接显示。当数据是 [1500000, 2000000, 1800000] 这种大数值时，库自动选最合适的中文 / 英文后缀缩放显示。

不用新增任何字段，全自动。

现场对比 ​

不缩放（小数值） ​

自动用「万」（百万级） ​

自动用「亿」（十亿级） ​

英文 K/M/B ​

阈值总表 ​

数据范围中文后缀英文后缀除数
< 1,000（无）（无）1
1,000 – 9,999—K10³
10,000 – 99,999,999万—10⁴
1,000,000 – 999,999,999—M10⁶
100,000,000+亿—10⁸
1,000,000,000+—B10⁹

库根据 unit 字段自动判断语境：包含「元」「￥」「$」等货币符走中文量级，纯英文（如 views）走 K/M/B。

精度策略 ​

缩放后数值小数位例
≥ 1000150万元
10 – 99115.5万元
1 – 921.55万元
< 130.155万

yPadding — 控制 Y 轴顶部留白 ​

如果觉得柱子顶到天花板，调节 yPadding：

值效果
0紧贴数据（最大柱子顶到 Y 轴顶端）
0.1（默认）留 10% 空白
0.3留 30%（适合演示、留空间放标注）
0.5留 50%（突出顶部留白）
js
// 默认 0.1（推荐）
new NovaChart('c', { type: 'bar', data: [100, 200, 150] }).draw();

// 留 30% 空白
new NovaChart('c', { type: 'bar', data: [100, 200, 150], yPadding: 0.3 }).draw();

// 紧贴
new NovaChart('c', { type: 'bar', data: [100, 200, 150], yPadding: 0 }).draw();

yPadding 只影响 bar 和 line。Table / Pie 不受影响（Table 显示绝对值，Pie 用百分比）。

不变原则 ​

  • ✅ 内部坐标计算仍用原始数值（柱子高度比例不变）
  • ✅ 只改显示文字（Y 轴刻度 / 柱顶数值 / Table 单元格）
  • ✅ 零新字段（yPadding 是唯一控制点，默认 0.1）
