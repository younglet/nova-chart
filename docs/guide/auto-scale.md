# 🔢 大数值场景

当数据是 [10, 50, 200] 这种小数值时，库直接显示。当数据是 [1500000, 2000000, 1800000] 这种大数值时，库**自动**选最合适的中文 / 英文后缀缩放显示。

不用新增任何字段，**全自动**。

## 现场对比

### 不缩放（小数值）

<NovaChart :config="{ type: 'bar', title: '期末成绩', labels: ['语文','数学','英语','物理','化学'], data: [85,92,78,88,90], unit: '分', theme: 'ocean' }" />

### 自动用「万」（百万级）

<NovaChart :config="{ type: 'bar', title: '月销售额（万元）', labels: ['1月','2月','3月','4月'], data: [1500000,2000000,1800000,2200000], unit: '元', theme: 'ocean' }" />

### 自动用「亿」（十亿级）

<NovaChart :config="{ type: 'line', title: '近五年营收', labels: ['2019','2020','2021','2022','2023'], data: [1200000000,1500000000,1800000000,2200000000,2600000000], unit: '元', theme: 'sunset' }" />

### 英文 K/M/B

## 阈值总表

| 数据范围 | 中文后缀 | 英文后缀 | 除数 |
|----------|----------|----------|------|
| < 1,000 | （无） | （无） | 1 |
| 1,000 – 9,999 | — | K | 10³ |
| 10,000 – 99,999,999 | 万 | — | 10⁴ |
| 1,000,000 – 999,999,999 | — | M | 10⁶ |
| 100,000,000+ | 亿 | — | 10⁸ |
| 1,000,000,000+ | — | B | 10⁹ |

> 库根据 `unit` 字段**自动判断**语境：包含「元」「￥」「$」等货币符走中文量级，纯英文（如 `views`）走 K/M/B。

## 精度策略

| 缩放后数值 | 小数位 | 例 |
|-----------|--------|------|
| ≥ 100 | 0 | `150万元` |
| 10 – 99 | 1 | `15.5万元` |
| 1 – 9 | 2 | `1.55万元` |
| < 1 | 3 | `0.155万` |

## `yPadding` — 控制 Y 轴顶部留白

如果觉得柱子顶到天花板，调节 `yPadding`：

| 值 | 效果 |
|----|------|
| `0` | 紧贴数据（最大柱子顶到 Y 轴顶端） |
| `0.1`（默认） | 留 10% 空白 |
| `0.3` | 留 30%（适合演示、留空间放标注） |
| `0.5` | 留 50%（突出顶部留白） |

```js
// 默认 0.1（推荐）
new NovaChart('c', { type: 'bar', data: [100, 200, 150] }).draw();

// 留 30% 空白
new NovaChart('c', { type: 'bar', data: [100, 200, 150], yPadding: 0.3 }).draw();

// 紧贴
new NovaChart('c', { type: 'bar', data: [100, 200, 150], yPadding: 0 }).draw();
```

> `yPadding` 只影响 **bar** 和 **line**。Table / Pie 不受影响（Table 显示绝对值，Pie 用百分比）。

## 不变原则

- ✅ 内部坐标计算仍用原始数值（柱子高度比例不变）
- ✅ 只改**显示文字**（Y 轴刻度 / 柱顶数值 / Table 单元格）
- ✅ 零新字段（`yPadding` 是唯一控制点，默认 0.1）
