# 为什么选择 NovaChart？

## 与 Chart.js 对比

| 维度 | Chart.js v4 | NovaChart |
|------|-------------|-----------|
| 源码行数 | 11,589 | **628** |
| 打包后体积 | ~200KB | **18KB** |
| API 字段数 | 30+ | **8** |
| 图表类型 | 8 种 | **4 种** |
| 坐标轴 | 6 种 | **2 种** |
| 主题 | 自行配置 | **3 套开箱** |
| 动画 | 完整引擎 | **3 种基础** |
| 学习成本 | 数小时 | **5 分钟** |

## 设计取舍

### 我们砍掉了什么？

| 砍掉的功能 | 理由 |
|-----------|------|
| Bubble / Scatter / Radar / PolarArea | 学生用不上 90% 的场景 |
| Doughnut（只保留 Pie） | Doughnut 视觉效果与 Pie 重复 |
| Logarithmic / Time / RadialLinear 坐标轴 | 学术场景，初学不需要 |
| 多数据集（datasets: [...]） | 引入后 API 复杂度翻倍 |
| 缩放 / 拖拽 / 平移 | 静态展示足够 |
| Tree-shaking 注册机制 | 学生不会按需引入 |
| TypeScript 类型定义 | v1 暂不提供 |
| 复杂动画引擎 | fade / slide 覆盖 95% 效果 |

### 我们保留了什么？

- ✅ 4 种最常用图表：Bar / Line / Pie / Table
- ✅ 2 种坐标轴：Category（分类）/ Linear（数值）
- ✅ 3 套主题：ocean / sunset / dark
- ✅ 3 种动画：fade / slide / none
- ✅ Canvas 高质量渲染（抗锯齿）
- ✅ 响应式：自适应父容器宽度
- ✅ 主题色联动：网格 / 文字 / 表格 / 图表

## 适合谁？

### 👍 推荐使用

- 🎓 **学生**：作业、毕设、网页作品
- 🐍 **Python 用户**：想在网页展示数据但不想学前端
- 🚀 **快速原型**：5 分钟内出图
- 📊 **静态报告**：不需要交互的可视化

### 👎 不推荐使用

- 💼 商业级数据大屏（用 ECharts / AntV）
- 🔬 复杂学术图表（用 D3 / Plotly）
- 📱 需要手势交互的移动端可视化
- 🏢 企业级 BI 工具（用 Highcharts）

## 一句话总结

> **如果你只想"画个图"，用 NovaChart。**  
> **如果你想"画好图"，用 Chart.js / ECharts。**  
> **如果你想"画奇葩的图"，用 D3。**
